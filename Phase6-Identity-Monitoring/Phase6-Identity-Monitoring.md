# Phase 6 – Identity Monitoring and Logging

## Objective

Implement Microsoft Entra monitoring capabilities to track authentication activity, administrative changes, and identity security events. Logging and monitoring allow security teams to detect suspicious behavior, investigate incidents, and maintain visibility into identity infrastructure.

This phase demonstrates how administrators monitor sign-ins, review audit events, evaluate Conditional Access decisions, and inspect identity risk signals within Microsoft Entra.

Navigation

Microsoft Entra Admin Center  
Identity Protection

The dashboard currently shows **no risky sign-ins detected**, which is expected in a controlled lab environment where no suspicious activity has occurred.

In production environments, this dashboard detects threats such as:

Impossible travel  
Anonymous IP sign-ins  
Password spray attacks  
Leaked credentials
